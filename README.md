# STM32F411 Bare-Metal GPIO & Floating Input Protection

Цей навчальний проєкт демонструє низькорівневе (Bare-Metal) програмування мікроконтролера STM32F411CE (Black Pill) на мові C без використання сторонніх бібліотек (HAL/LL). 

Програма реалізує алгоритм опитування (Polling) шляхом безперервного зчитування стану вхідного піна PA0 та керує вбудованим світлодіодом на PC13.
У коді враховано інверсну логіку роботи світлодіода (Active LOW) та реалізовано апаратний захист від електромагнітних завад ("плаваючого входу").

## Особливості проєкту
- Регістрове керування: Пряма маніпуляція регістрами RCC_AHB1ENR, MODER, PUPDR, IDR, ODR через вказівники на базові адреси периферії.
- Захист від завад (Floating Input): Конфігурація внутрішнього стягувального резистора (Pull-down до землі), що усуває ефект антени на відкритому дроті.
- Адаптація под Black Pill: Реалізовано інверсну логіку для вбудованого синього світлодіода.
- Чистота коду: Оптимізовано тактування портів за один крок (Read-Modify-Write), враховано пріоритети бітових операцій шляхом явного маскування дужками.

## Апаратне забезпечення та розпіновка
- Мікроконтролер: STM32F411CEU6 (Black Pill)
- Дріт-перемичка (Dupont) для ручної зміни потенціалу на вході.

| Пін МК  | Функція            | Конфігурація регістрів                           | Опис логіки                                                    |
| :------ | :----------------- | :----------------------------------------------- | :------------------------------------------------------------- |
| PA0     | Вхід (Дріт-тестер) | MODER = 0b00 (Input)<br>PUPDR = 0b10 (Pull-down) | Захищений від наведень. Читає чіткий HIGH (3.3V) або LOW (GND) |
| PC13    | Вихід (Onboard LED)| MODER = 0b01 (Output)                            | Active LOW: логічний 0 вмикає світлодіод, логічна 1 - вимикає  |

## Алгоритм роботи
1. Увімкнення тактування портів GPIOA та GPIOC за одну операцію запису в RCC_AHB1ENR.
2. Налаштування апаратної підтяжки Pull-down для усунення плаваючого стану на PA0 (запис 0b10 в PUPDR).
3. Явне скидання бітів режиму PA0 в інпут (0b00 в MODER).
4. Налаштування PC13 в режим виходу загального призначення (0b01 в MODER).
5. У нескінченному циклі while(1) виконується побітова перевірка стану регістру GPIOA_IDR:
   - Якщо на PA0 виявлено 3.3V (HIGH) -> очищується 13-й біт GPIOC_ODR -> LED вмикається.
   - Якщо на PA0 виявлено GND (LOW) -> встановлюється 13-й біт GPIOC_ODR -> LED вимикається.

##  Результати тестування

### 1. Подача високого рівня (HIGH) на PA0
Коли на вхідний пін подано напругу 3.3V, мікроконтролер фіксує рівень HIGH та встановлює на виході PC13 низький рівень (0V), що призводить до ввімкнення світлодіода.

### 2. Подача низького рівня (LOW) на PA0
Коли на вхідний пін подано потенціал землі GND (або пін залишено в повітрі завдяки активованому Pull-down резистору),
мікроконтролер фіксує рівень LOW та встановлює на виході PC13 високий рівень (3.3V), що призводить до вимкнення світлодіода.
